# routes/

TanStack Router file-based routes. Route tree is auto-generated by `@tanstack/router-plugin` into `routeTree.gen.ts`.

## Routing Convention

File names map to URL segments using dot notation:
- `resource.$resourceType.tsx` -> `/resource/:resourceType`
- `resource.ViewDefinition.edit.$id.tsx` -> `/resource/ViewDefinition/edit/:id`

All routes are served under base path `/u` (configured in `src/index.tsx`).

## Route Files

| File | URL | Description |
|------|-----|-------------|
| `__root.tsx` | - | Root layout, wraps all routes with `<Layout>` + devtools |
| `index.tsx` | `/` | Home page (empty placeholder) |
| `rest.tsx` | `/rest` | REST Console - full HTTP client with tabs, collections, history |
| `resource.tsx` | `/resource` | Parent layout for all resource routes (just renders `<Outlet />`) |
| `resource.index.tsx` | `/resource` | Resource type index |
| `resource.$resourceType.tsx` | `/resource/:type` | Parent for dynamic resource type |
| `resource.$resourceType.index.tsx` | `/resource/:type` | Resource Browser page |
| `resource.$resourceType.create.tsx` | `/resource/:type/create` | Create new resource |
| `resource.$resourceType.edit.$id.tsx` | `/resource/:type/edit/:id` | Edit resource |
| `resource.ViewDefinition.tsx` | `/resource/ViewDefinition` | Parent for ViewDefinition routes |
| `resource.ViewDefinition.index.tsx` | `/resource/ViewDefinition` | ViewDefinition Browser |
| `resource.ViewDefinition.create.tsx` | `/resource/ViewDefinition/create` | Create ViewDefinition |
| `resource.ViewDefinition.edit.$id.tsx` | `/resource/ViewDefinition/edit/:id` | Edit ViewDefinition |

## Patterns

- Each route exports `Route` created via `createFileRoute(path)` with `component`, `staticData.title`, and optional `loader`
- Search params are used for tab state and editor mode (e.g., `?tab=code&mode=json`)
- Route components typically instantiate `useAidboxClient()` and pass it to feature components in `src/components/`
